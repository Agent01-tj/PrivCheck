Learn how to use PrivCheck effectively

What is PrivCheck?

PrivCheck is a Windows privilege escalation auditor designed to identify dangerous privileges that could lead to system compromise. It's a self-contained batch script that requires no dependencies and provides immediate security insights.

Why do I need to run it as Administrator?

Administrator privileges are required to:

  • Access complete privilege information for the current user
  • View all system-level security contexts
  • Generate comprehensive audit reports
  • Ensure accurate risk assessment

What privileges are considered dangerous?

PrivCheck monitors these high-risk privileges:

  • SeDebugPrivilege: Allows debugging of processes and direct memory access
  • SeImpersonatePrivilege: Enables impersonation of other users' security contexts
  • SeAssignPrimaryTokenPrivilege: Allows replacement of process tokens
  • SeTakeOwnershipPrivilege: Permits taking ownership of system objects
  • SeBackupPrivilege: Enables reading files while bypassing ACLs
  • SeRestorePrivilege: Allows writing files while bypassing ACLs

How do I interpret the risk levels?

LOW 1 dangerous privilege found - Monitor and review
MEDIUM 2-3 dangerous privileges found - Immediate attention required
HIGH (CRITICAL) 4+ dangerous privileges found - Critical security risk

How to remediate dangerous privileges?

  1. Open Local Security Policy (Start ‚Üí Run ‚Üí secpol.msc)
  2. Navigate to: Security Settings ‚Üí Local Policies ‚Üí User Rights Assignment
  3. For each dangerous privilege found:
    • Double-click the privilege name
    • Remove unnecessary user accounts
    • Keep only essential system accounts
  4. Apply changes and restart if required
  5. Re-run PrivCheck to verify remediation

Best Practices

  • Regular Audits: Run PrivCheck monthly or after system changes
  • Principle of Least Privilege: Only grant necessary privileges
  • Service Account Review: Regularly audit service account privileges
  • Documentation: Keep records of privilege changes and justifications
  • Monitoring: Set up alerts for privilege modifications

Troubleshooting

Script not running as administrator warning

Right-click the batch file and select "Run as administrator" to ensure full privilege visibility.

No output or empty report

Ensure you have proper permissions and the script is running on a Windows system with the whoami command available.

False positives

Some legitimate system accounts may require certain privileges. Review each privilege's necessity for your specific environment.
